# AI Development Guidelines

## Project Context

This repository contains an AI-assisted frontend capstone project developed for the FLYRANK AI Frontend Internship.

## Technology Stack

- React
- TypeScript
- Vite
- Node.js
- CSS
- Git and GitHub

## Code Conventions

- Use TypeScript for application code.
- Prefer functional React components.
- Keep components small and focused.
- Use clear and descriptive names.
- Avoid duplicated logic.
- Use semantic HTML.
- Ensure the application is responsive and accessible.

## Git Conventions

All commits must follow Conventional Commits.

Examples:

- `feat: add search interface`
- `fix: correct mobile navigation`
- `docs: update setup instructions`
- `refactor: extract reusable component`
- `test: add component tests`
- `chore: configure project tooling`

Each commit should represent one logical change.

## AI Assistant Instructions

1. Review the existing project structure before editing files.
2. Explain planned changes briefly.
3. Make the smallest reasonable change.
4. Preserve existing functionality.
5. Avoid adding dependencies without justification.
6. Check TypeScript, accessibility, and responsive-design concerns.
7. Review the final changes before suggesting a commit.
8. Suggest a Conventional Commit message.
9. Do not claim code was tested unless the relevant command was run.
